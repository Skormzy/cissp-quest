## What this project is

**CISSP Quest** is a gamified CISSP certification learning platform. NOT just a quiz app. Three layers stacked:
1. **Reference Library** - clear explanations with analogies, always accessible from nav
2. **Story Lessons** - anime visual-novel style narrative ("Operation Shadow Protocol") across 8 global locations, one per CISSP domain
3. **Quiz Engine** - CAT (Computerized Adaptive Testing) using IRT scoring

**Core principle:** TEACH FIRST, THEN TEST. Wrong answers always link back to a Library topic with a Memory Hack reminder.

**Target user:** CISSP candidates (Seymour himself and eventually paid users) who want a Duolingo-style alternative to dry textbook study.

**Success criteria ("done" looks like):**
- All 8 CISSP domains covered with original content
- Stripe payments live and secure
- Free tier (Domain 1) unlocked publicly
- Story-mode and quiz-engine stable on mobile and desktop
- Content quality strong enough that Seymour himself uses it for exam prep

**Current status:** Active build, PRE-LAUNCH, not shipping to customers yet.

**Classification:** personal SaaS (Seymour's flagship personal product, stays on personal PC)

## Architecture

### File organization
- `src/app/` - Next.js App Router pages and API routes
  - `src/app/(auth)/` - auth route group (login, signup)
  - `src/app/app/` - authenticated app routes (dashboard, quiz, story, library, tools, community, upgrade)
  - `src/app/api/stripe/` - Stripe checkout and webhook routes
- `src/components/` - composite components organized by domain (audio, character, community, dashboard, layout, medals, nav, quiz, story, tools, ui, visual-novel)
- `src/components/ui/` - shadcn primitives (do not put composites here)
- `src/lib/` - business logic
  - `cat-session.ts`, `irt-engine.ts` - adaptive testing engine (IRT = Item Response Theory)
  - `story-content.ts`, `ghost-pov-content.ts` - narrative content
  - `stripe/config.ts` - Stripe config
  - `audio/ambient-audio.ts` - Tone.js wrapper
- `src/stores/` - Zustand stores, one per domain
- `src/data/` - static content (story acts, glossary)
- `scripts/` - one-off ingestion/generation scripts (excluded from tsconfig)
- `source-material/` - PDFs and extracted content (gitignored)
- `supabase-MASTER-migration.sql` at root - applied via Supabase MCP or dashboard

Practice exam uses CAT with IRT scoring, mirroring the real CISSP exam.

### The 8 CISSP domains (all must be covered)

1. Security and Risk Management (15%)
2. Asset Security (10%)

4. Communication and Network Security (13%)
5. Identity and Access Management (13%)

### Story: Operation Shadow Protocol

One shared narrative across all 8 domains (stronger narrative encoding than 8 separate mini-stories). Recurring NPCs reinforce concepts across domains. Memory techniques embedded: Method of Loci, mnemonics, spaced repetition, chunking, elaborative interrogation, dual coding.

Story assets (85 webp images): backgrounds per act, NPC character portraits with emotional variants (Tanaka, Ghost, Sharma, Vasquez, Webb), concept diagrams (CIA triad, Bell-LaPadula, Biba, kill chain, OSI, etc.), evidence artifacts, UI elements.

### Content philosophy (IMPORTANT, do not violate)

- **Original content only.** No copy-paste from textbooks or existing question banks. Informed by ISC2 outline and reference textbooks, written fresh.
- **Voice:** "Think Like a Manager" - strategic, not tactical
- **Memory hack per concept:** every Library entry and question gets a mnemonic, acronym, or location anchor
- **Scenario stems:** questions use BEST/MOST/FIRST framing with 4 plausible options

## Critical known issues (DO NOT ship Stripe without fixing these)

### Critical (3)
1. **Missing Stripe DB columns** - `stripe_customer_id`, `stripe_subscription_id`, `subscription_expires_at` referenced in code but not in migration. Users pay, don't get access.
2. **RLS allows self-upgrade** - anyone can open browser console and set themselves to expert tier
3. **Webhook signature verification optional** - falls back to plaintext parse if STRIPE_WEBHOOK_SECRET missing

### High (4)
4. **Webhook DB failures silently return 200** - Stripe wont retry, failed updates lost
5. **Paid-feature gates client-only** - Ghost POV, Evidence Constellation, Practice Exam all bypassable via URL
6. **Community tables missing from migration** - chatroom_messages, weekly_scores, notification_queue referenced but no CREATE TABLE
7. **Annual billing UI ignored** - parameter name mismatch, everyone gets monthly

### Medium (5)
8. success_url trusts request Origin header
9. Subscription lifecycle doesnt check status (past-due retains access)
10. Practice exam domain weights not enforced in selectNext()
11. SPRT helper pass/fail inverted in irt-engine.ts
12. seed-questions.ts destructive and incomplete

**Priority order for fixing** (documented in REVIEW_FINDINGS_2026-04-19.md):
1. Add missing DB columns and migration (fixes #1, #6)
2. Lock down RLS (fixes #2)
3. Require webhook signature verification (fixes #3)
4. Check tier server-side on paid routes (fixes #5)
5. Fix webhook error handling (fixes #4)
6. Fix annual billing end-to-end (fixes #7)
7. Fix success_url origin validation (fixes #8)
8. Add subscription status check (fixes #9)
9. Fix SPRT inversion (fixes #11)
10. Fix seed script and domain weights (fixes #10, #12)
